# Gigabrain v0.6.0

## Summary

`v0.6.0` is the first Gigabrain release that treats recall quality as something we can measure, not just tune by feel.

This release hardens production maintenance, adds a real eval harness, upgrades the recall stack with BM25 plus config-gated semantic reranking, exposes more of the world model directly to Codex and Claude through MCP, and closes the final stable-release gaps around OpenClaw hooks, empty-store bootstrap, install helpers, runtime guards, and Claude Desktop launcher reliability. It also resolves the three public OpenClaw correctness issues reported in `#40`, `#41`, and `#42`.

## Highlights

- **Production hardening**
  - capture parsing now strips model thinking blocks before note parsing
  - semantic dedupe timeout resolution now archives the verified loser and keeps auditable queue metadata
  - nightly artifacts now include recall latency summaries, graph node/edge counts, and explicit FTS rebuild status
  - review/audit idempotency now fingerprints the effective config so changed thresholds do not get skipped under the same review version
- **Recall evaluation**
  - new `eval-harness.js` powers `scripts/eval-runner.js` and `scripts/eval-compare.js`
  - nightly maintenance now persists `memory_eval_history` rows with canonical metrics like `precision_at_3`, `mrr`, `hit_rate`, `latency_median_ms`, and `latency_p95_ms`
- **Smarter recall**
  - BM25 replaces the earlier overlap-only lexical ranking
  - adaptive budgets and confidence labels make recall output more strategy-aware
  - semantic rerank is available behind config and degrades cleanly to lexical-only when Ollama is unavailable
- **Stable host integrations**
  - OpenClaw recall hooks now inject via `appendSystemContext`, merge `ctx`, and keep scope isolation instead of silently falling back to `shared`
  - zero-memory sessions now still receive a minimal `<gigabrain-context>` block with `bootstrap_mode: true`, so first-run installs do not deadlock
  - Codex and Claude setup helpers now resolve Gigabrain dynamically from repo-local binaries, host binaries, or `npx --no-install` instead of depending on setup-time absolute paths
  - Claude Desktop bundles now launch through a bundled `/bin/sh` wrapper that prepends common macOS/Homebrew PATHs before `exec node`
- **World-model surfaces**
  - `/gb/evolution` and `/gb/relationships` are now first-class HTTP routes
  - standalone MCP now exposes `gigabrain_entity`, `gigabrain_contradictions`, and `gigabrain_relationships`
  - entity evolution, contradiction suggestions, and relationship views now share normalized helper outputs across HTTP, MCP, and vault surfaces

## Upgrade Notes

- Existing `0.5.3` standalone installs remain compatible.
- Fresh standalone installs continue to default to `~/.gigabrain/config.json`, while supported legacy installs under `~/.codex/gigabrain/config.json` are reused in place.
- Upgrading from `v0.5.x` should include one rerun of `gigabrain-codex-setup` or `gigabrain-claude-setup` so helper scripts, shared user-store wiring, and launcher defaults are refreshed together.
- Claude Desktop portable bundles assume setup has already created the standalone config. Fresh users should run `gigabrain-claude-setup` before importing the `.dxt`; legacy users should use the resolved setup path if Claude prompts for one.
- Gigabrain now fails fast with a friendly error on Node `<22` instead of surfacing raw `node:sqlite` import failures.
- Semantic rerank is still **off by default** and does not change behavior for users without Ollama.
- OpenClaw, Codex, Claude Code, and Claude Desktop continue to use the same local-first store model introduced in `0.5.3`.
- Repo scopes still derive from `project:<repo>:<hash>` using the absolute repo path on the current machine; moving a repo to a different path or machine will produce a new default scope unless you pin it explicitly.
