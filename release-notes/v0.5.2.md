# Gigabrain v0.5.2

## Summary

`v0.5.2` is a reliability patch release built on top of `v0.5.1`.

It keeps the `v0.5.1` Codex standalone improvements and adds the two production fixes that were still needed afterward: safer SQLite startup behavior inside the plugin entrypoint, and built-in nightly run protection in the control CLI.

## Highlights

### Startup reliability

Gigabrain's OpenClaw entrypoint now uses the shared SQLite opener instead of opening the registry directly without wait handling.

That means transient writer contention now respects SQLite `busy_timeout` instead of failing immediately with `database is locked` during plugin register or startup.

### Nightly reliability

`gigabrainctl nightly` now includes its own operational guardrails:

- output-scoped exclusive lock for each nightly run
- stale dead-owner lock cleanup
- clean JSON skip when another nightly run is already active
- post-run verification of the execution artifact and usage log before success is reported

This moves the critical reliability behavior into the product path instead of leaving it only in Nimbus host wrappers.

## Validation

Validated for this patch release on **March 13, 2026**:

- GitHub `v0.5.1` used as the release base
- current suite passing via `npm run test:unit`
- dedicated nightly CLI integration coverage passing
- live Nimbus nightly run completing with `verification.ok=true`

## Scope note

`v0.5.2` is intentionally narrow. It preserves the `v0.5.1` standalone Codex work and does not change schemas, APIs, or the world-model feature surface.
